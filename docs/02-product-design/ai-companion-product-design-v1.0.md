## 1. 产品定位

### 1.1 一句话定位

> **"每个孩子身边的AI专属教师,每个家长手中的学习透视镜"**

### 1.2 产品愿景

通过AI Agent赋能K12教育，为每个孩子提供个性化的AI专属教师，同时增强家长对孩子学习过程的透明度和参与感。AI不是冷冰冰的题库工具，而是真正了解孩子、因材施教的智能教师。

### 1.3 目标用户

| 角色 | 画像 | 核心诉求 |
|------|------|---------|
| **家长** | 小学+初中阶段（1-9年级，6-15岁）孩子的父母 | 了解孩子学习情况、有效参与孩子学习、看到真实进步 |
| **学生** | 小学1年级 ~ 初中9年级（6-15岁） | 有趣的学习体验、即时反馈鼓励、成就感 |

### 1.4 支持学科

| 学科 | 核心教学目标 | 发布阶段 |
|------|------------|---------|
| **英语** | 词汇记忆、发音练习、情景运用、阅读理解、口语对话 | Phase 1 (MVP) |
| **语文** | 经典文化阅读、理解能力、表达能力 | Phase 2 |
| **数学** | 概念理解、练习巩固、数学思维培养 | Phase 3 |

## 2. 核心价值主张

### 2.1 价值三角模型

```
              ┌─────────────┐
              │   AI 教师    │
              │  (智能中枢)   │
              └──────┬──────┘
                     │
            ┌────────┴────────┐
            │  全人化记忆引擎   │
            │  动态学习路径规划  │
            │  多模态教学交互   │
            └────────┬────────┘
                     │
         ┌───────────┼───────────┐
         ▼                       ▼
  ┌─────────────┐         ┌─────────────┐
  │   家长端     │         │   孩子端     │
  │  (决策+透明)  │         │  (学习+互动)  │
  ├─────────────┤         ├─────────────┤
  │• 审批学习内容 │         │• 与AI教师互动  │
  │• 设置时间窗口 │         │• 语音/文字/拍照│
  │• 即时学习简报 │         │• 游戏化激励    │
  │• 管理多个孩子 │         │• 即时反馈鼓励  │
  └─────────────┘         └─────────────┘
```

### 2.2 核心商业逻辑闭环

```
家长订阅 → AI建档 → AI智能推荐学习内容
    ↑                        ↓
    │                  家长审批确认
    │                        ↓
 价值感知 ←── 即时简报 ←── 孩子与AI教师互动学习
    │                        ↓
    └──── 孩子进步可见 ←── AI更新全人档案 ──→ 更精准的推荐
```

**飞轮效应**: 孩子用得越多 → AI越了解孩子 → 推荐越精准 → 学习效果越好 → 家长越愿意续费 → 迁移成本越高

### 2.3 与市场现有产品的差异化

| 维度 | 市面产品（百词斩/猿题库等） | 本产品 |
|------|--------------------------|--------|
| 记忆 | 只记录做题数据 | **全人化记忆**（性格、风格、兴趣、习惯） |
| 路径 | 固定课程体系 | **AI动态规划**，千人千面 |
| 家长 | 被动查看成绩 | **主动审批+智能建议**，深度参与 |
| 互动 | 选择题为主 | **多模态**（语音、对话、拍照、表达） |
| 激励 | 简单积分 | **游戏化体系**融入学习全程 |
| 推送 | 用户自己打开 | **AI智能推送**，基于记忆曲线和最佳时机 |

## 3. 商业模式

### 3.1 收费模式

**订阅制 SaaS**，按学科独立订阅。

- 家长按月/季/年订阅，可自由选择订阅1-3个学科
- 每个学科独立定价
- 多孩家庭可享受家庭套餐优惠（后续设计）

### 3.2 账户体系

**家庭账户制**:

```
┌─────────────────────────────────┐
│         家庭账户                 │
│  ┌─────────┐  ┌─────────┐      │
│  │ 家长 A   │  │ 家长 B   │     │
│  │ (父/母)  │  │ (父/母)  │     │
│  └────┬─────┘  └────┬────┘     │
│       │              │          │
│  ┌────┴──────────────┴────┐    │
│  │       孩子管理          │    │
│  │  ┌──────┐  ┌──────┐   │    │
│  │  │孩子 A │  │孩子 B │   │    │
│  │  │独立档案│  │独立档案│   │    │
│  │  │独立订阅│  │独立订阅│   │    │
│  │  └──────┘  └──────┘   │    │
```

- 一个家庭账户支持父母双方登录管理
- 每个孩子独立建档、独立学习记录、独立订阅
- 父母双方均可审批内容、查看学习报告

## 4. 核心功能模块设计

### 4.1 用户注册与孩子建档

**家长注册流程**:
1. 手机号/微信注册 → 创建家庭账户
2. 添加孩子基本信息:
   - 姓名、性别、头像
   - **年龄和K12阶段**（最关键字段，决定内容难度和AI交互风格）
   - 就读学校/年级（可选）
   - 兴趣爱好（可选，用于AI个性化）
3. 邀请另一位家长加入家庭账户（可选）

**电子档案内容（AI全人化记忆）**:

| 记忆层次 | 具体内容 | 更新机制 |
|---------|---------|---------|
| **基础信息** | 年龄、年级、K12阶段 | 家长手动设置 |
| **学习进度** | 各科知识点掌握情况、历史正确率、学习时长 | 每次学习后自动更新 |
| **学习风格** | 视觉型/听觉型/动手型、注意力持续时长、最佳学习时段 | AI通过互动数据分析推断 |
| **性格特点** | 内向/外向、抗挫折能力、竞争意识、自主性 | AI通过互动反应逐步建模 |
| **兴趣偏好** | 喜欢的主题（动物/太空/历史等）、喜欢的互动方式 | AI观察+家长补充 |
| **个性化策略** | 最有效的鼓励方式、适合的教学节奏、薄弱环节突破策略 | AI持续优化 |

### 4.2 AI智能推送机制

**核心流程（三科通用）**:

```
AI分析引擎                    家长端                    孩子端
    │                          │                         │
    ├── 分析孩子当前学习状态     │                         │
    ├── 结合艾宾浩斯记忆曲线     │                         │
    ├── 匹配K12阶段内容库       │                         │
    ├── 生成推送方案 ──────────→│                         │
    │   包含:                   │                         │
    │   • 当前学习情况总结       ├── 查看推送详情           │
    │   • 本次推送缘由说明       ├── 确认/修改/拒绝         │
    │   • 预期达到效果          │                         │
    │                          ├── 确认推送 ─────────────→│
    │                          │                         ├── 接收学习任务
    │                          │                         ├── 与AI教师互动
    │                          │                         ├── 完成学习
    │                          │            即时简报 ←────┤
    │←──── 更新孩子档案 ────────┤                         │
    │                          │                         │
```

**推送时机**: AI智能时间规划
- 家长设置"可学习时间窗口"（如：工作日 16:00-20:00，周末 9:00-11:00 / 14:00-17:00）
- AI在窗口内根据以下因素决定最佳推送时机:
  - 艾宾浩斯记忆曲线（英语单词复习最佳时间点）
  - 孩子的注意力规律（AI学习到的最佳学习时段）
  - 学习负荷平衡（避免连续高强度）
  - 学科间的合理穿插

**家长审批推送内容包含**:
1. **学习情况总结**: "小明本周英语学习了45个新单词，掌握率82%，'动物类'词汇较弱"
2. **本次推送缘由**: "根据记忆曲线，3天前学习的15个动物类单词需要第一次复习，同时推荐5个新单词"
3. **预期效果**: "完成本次学习后，预计动物类词汇掌握率提升至90%，总词汇量达到50个"
4. **操作选项**: 确认推送 / 调整内容 / 延后推送

### 4.3 家长端核心功能

| 功能模块 | 说明 |
|---------|------|
| **家庭仪表盘** | 所有孩子的学习状态概览，一目了然 |
| **审批中心** | 查看AI推荐的学习内容，确认/修改/拒绝 |
| **即时简报** | 每次孩子学习后收到简要反馈（正确率、进步点、需加强点） |
| **学习设置** | 设置可学习时间窗口、学科偏好、学习强度 |
| **孩子管理** | 添加/管理孩子档案、查看各孩子详细学习数据 |
| **订阅管理** | 管理各孩子的学科订阅 |

### 4.4 孩子端核心功能

| 功能模块 | 说明 |
|---------|------|
| **学习任务** | 接收家长确认的学习任务，清晰的任务列表 |
| **AI教师互动** | 多模态互动学习（语音、文字、拍照） |
| **即时反馈** | 每个回答后AI给予对错判断+指导+鼓励 |
| **成就中心** | 积分、徽章、等级、连续打卡记录 |
| **学习记录** | 查看自己的学习历史和进步轨迹 |

## 5. 三大学科详细设计

### 5.1 英语学科（Phase 1 MVP）

**核心教学法**: 艾宾浩斯21天记忆法 + AI个性化教学

#### 五大学习场景

**场景一: 艾宾浩斯单词记忆**

学习流程:
- 新单词学习 → 5分钟后第1次复习 → 30分钟后第2次 → 12小时后第3次
- → 1天后第4次 → 2天后第5次 → 4天后第6次 → 7天后第7次 → 15天后第8次

AI介入点:
- 根据孩子K12阶段匹配词汇库（小学800词 → 初中1600词 → 对标课标）
- 每次复习时根据孩子遗忘模式动态调整间隔
- 针对弱项词汇增加复习频率
- 用孩子感兴趣的主题组织词汇组

互动方式:
- AI展示单词 → 孩子选择/输入中文含义
- AI报读英文 → 孩子跟读/拼写
- AI用单词造句 → 孩子理解判断
- 图片联想记忆 → 孩子看图说单词

**场景二: 发音/语音练习**
- AI标准发音示范 → 孩子跟读
- AI语音识别评估发音准确度（音标级别）
- 连读、弱读、重音等进阶练习（初中阶段）
- 实时纠音 + 对比回放

**场景三: 情景化造句/对话**
- 将当前学习的单词放入生活场景
- AI引导孩子用新学的单词造句
- 渐进式难度：填空 → 半开放造句 → 自由表达

**场景四: 英语分级阅读**
- 根据孩子词汇量和理解力匹配阅读材料
- AI朗读文章 → 孩子跟读
- 阅读理解提问 → 检验理解深度
- 生词自动纳入艾宾浩斯复习体系

**场景五: 主题式口语对话**
- 基于主题（家庭、学校、旅行、动物等）开展对话
- AI扮演不同角色与孩子对话练习
- 从简单问答到自由对话，逐步提升
- 语法纠正融入对话过程中

### 5.2 语文学科（Phase 2）

**核心教学目标**: 理解能力 + 表达能力

**素材来源**: 中国古典典籍、历史文化、成语故事、古诗词、文言文

**学习场景**: 经典文化阅读 + 表达训练

学习流程:
1. AI选取适合孩子阶段的文章/诗词
2. 家长审批
3. 推送至孩子端
4. 第一阶段: 聆听与阅读（AI朗读、孩子跟读、生僻字讲解）
5. 第二阶段: 理解与互动（AI提问检验理解）
6. 第三阶段: 表达与总结（孩子复述、AI评估表达）

### 5.3 数学学科（Phase 3）

**核心教学目标**: 概念理解 + 练习巩固 + 思维培养

**三大学习场景**:
1. **概念教学**: AI用孩子感兴趣的方式讲解数学概念
2. **练习巩固**: AI智能出题、拍照批改、错题引导
3. **思维挑战**: 每日趣味思维题，培养数学思维

## 6. AI教师核心能力设计

### 6.1 全人化记忆引擎

AI教师对每个孩子维护一个持续更新的"全人画像"：

```
┌─────────────────────────────────────┐
│          孩子全人画像                 │
├─────────────────────────────────────┤
│  📊 学习数据层                       │
│  ├── 各科知识点掌握度               │
│  ├── 历史正确率趋势                 │
│  ├── 学习时长与频率                 │
│  └── 薄弱知识点图谱                 │
│                                     │
│  🧠 认知风格层                       │
│  ├── 学习风格（视觉/听觉/动手）     │
│  ├── 注意力模式（持续时长/最佳时段） │
│  ├── 理解速度（快速/稳步）          │
│  └── 记忆特点                       │
│                                     │
│  💛 性格情感层                       │
│  ├── 性格特征（内向/外向/好奇心等） │
│  ├── 抗挫折能力                     │
│  ├── 最有效的鼓励方式               │
│  └── 兴趣爱好与话题偏好             │
│                                     │
│  🎯 教学策略层                       │
│  ├── 最佳教学节奏                   │
│  ├── 薄弱环节突破策略               │
│  ├── 已验证有效的教学方法           │
│  └── 个性化交互风格                 │
└─────────────────────────────────────┘
```

### 6.2 AI教师交互风格

AI教师根据孩子的年龄和性格自动调整交互风格：

| 年龄段 | 语言风格 | 互动特点 |
|--------|---------|---------|
| 6-8岁（低年级） | 亲切、简单、充满趣味，多用比喻和故事 | 高鼓励频率、游戏化互动、短时高频 |
| 9-11岁（中年级） | 友好、清晰、适当挑战 | 适度鼓励+引导思考、逐步增加自主性 |
| 12-15岁（初中） | 尊重、理性、像朋友一样对话 | 启发式教学、鼓励独立思考、接受挑战 |

### 6.3 AI动态学习路径规划

输入因素:
- 孩子当前各知识点掌握度
- 艾宾浩斯记忆曲线数据
- K12课标要求
- 孩子的学习风格和速度
- 近期学习状态（进步/平稳/退步）
- 家长设置的时间窗口和学习强度

输出:
- 下一步学什么（知识点选择）
- 用什么方式学（教学方法）
- 什么时候学（推送时机）
- 学多少（学习量控制）
- 给家长的推荐理由和预期效果

## 7. 游戏化激励体系

### 7.1 激励元素

| 元素 | 说明 | 目的 |
|------|------|------|
| **积分** | 每次学习获得积分，答对加分，连续学习有加成 | 量化学习投入 |
| **徽章** | 完成里程碑解锁徽章（如"单词大师100""阅读达人"） | 成就感标记 |
| **等级** | 从"学习新手"到"学霸"的等级体系 | 长期成长感 |
| **连续打卡** | 连续学习天数记录，中断后可用积分"补签" | 培养习惯 |
| **每日挑战** | 每天一道特殊挑战题，完成获额外奖励 | 保持新鲜感 |

### 7.2 激励与学习的融合原则

- 激励服务于学习，不喧宾夺主
- 重视"过程激励"而非仅"结果激励"（努力尝试也获得鼓励）
- 避免过度竞争，强调"和昨天的自己比"
- 家长可在简报中看到孩子的成就进展

## 8. 产品发布策略

### 8.1 英语先行，逐科验证

```
Phase 1 (M1-M3): 英语 MVP
  ├─ 家庭账户体系
  ├─ 孩子电子建档
  ├─ 英语全场景 (5个子场景)
  ├─ 家长审批流 + 即时简报
  ├─ AI全人化记忆
  └─ 游戏化激励 v1

Phase 2 (M4-M5): 语文扩展
  ├─ 复用英语的核心架构
  └─ 语文经典阅读+表达

Phase 3 (M6-M7): 数学扩展
  ├─ 数学概念教学+思维挑战
  └─ 拍照识别批改
```

### 8.2 选择英语先行的理由

1. **艾宾浩斯记忆法最标准化**: 有明确的科学依据和算法模型
2. **AI交互最成熟**: 语音识别、发音评估、对话能力已有成熟方案
3. **家长付费意愿强**: 英语学习是K12家长最舍得投入的学科
4. **效果可量化**: 单词量、正确率等指标让家长直观看到进步
5. **验证核心架构**: 通用能力可在英语中完整验证

## 9. 关键设计决策记录

| # | 决策项 | 决策结果 | 决策理由 |
|---|--------|---------|---------|
| 1 | 商业模式 | 订阅制 SaaS | 现金流稳定，用户粘性高 |
| 2 | 订阅粒度 | 单科选择制 | 给家长最大灵活性 |
| 3 | 目标用户 | 小学+初中（1-9年级） | 覆盖家长参与度最高的阶段 |
