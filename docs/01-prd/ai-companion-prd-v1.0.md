## 1. 产品概述

### 1.1 产品定位

AI智慧学伴是一款面向K12学生（1-9年级）和家长的教育类微信小程序，通过AI Agent技术为每个孩子提供个性化的AI专属教师，同时增强家长对孩子学习过程的透明度和参与感。

**一句话定位**: 每个孩子身边的AI专属教师，每个家长手中的学习透视镜。

### 1.2 产品目标

**Phase 1 目标（英语MVP）**:
- 验证"AI教师+家长监督+孩子学习"三角模型的商业可行性
- 建立核心的AI全人化记忆引擎和个性化教学能力
- 完成英语学科的完整学习闭环
- 获得首批种子用户并验证付费意愿

**长期目标**:
- 成为K12家庭首选的AI教育助手
- 覆盖语文、数学、英语三大主科
- 建立行业领先的AI个性化教学能力

### 1.3 核心价值主张

| 用户角色 | 核心痛点 | 我们的解决方案 | 核心价值 |
|---------|---------|--------------|---------|
| **家长** | 不了解孩子学习情况、无法有效参与、担心学习效果 | 智能推送建议+审批机制+即时简报 | 透明+控制+省心 |
| **孩子** | 学习枯燥、缺乏个性化、没有成就感 | AI专属教师+多模态互动+游戏化激励 | 有趣+有效+有成就感 |

**差异化优势**:
1. **全人化记忆**: AI不只记录学习数据，还了解孩子性格、兴趣、学习风格
2. **智能平衡**: AI处理日常决策，家长控制关键节点
3. **多模态交互**: 语音、文字、拍照，贴合真实学习场景
4. **动态路径**: 千人千面的学习路径，而非固定课程

**家长（决策者/付费者）**

| 属性 | 描述 |

| 家庭结构 | 双职工家庭，可能有1-3个孩子 |

| 属性 | 描述 |

### 2.2 典型使用场景

**场景1: 工作日晚上的学习时间**
- 18:00 家长下班回家，打开小程序查看今日待审批推送
- 18:15 家长审批通过AI推荐的10个英语单词学习任务
- 18:30 孩子吃完晚饭，打开小程序看到新任务
- 18:35-18:55 孩子与AI教师互动学习20分钟
- 19:00 家长收到即时简报："小明今天学了10个新单词，正确率90%，进步明显！"

**场景2: 周末的深度学习**
- 09:00 AI根据艾宾浩斯曲线推送复习任务
- 09:15 家长审批后，孩子开始复习本周学过的单词
- 09:45 完成单词复习后，AI推荐分级阅读文章
- 10:00-10:30 孩子阅读英文短文，AI提问检验理解
- 10:35 孩子解锁"阅读达人"徽章，获得50积分

**场景3: 碎片时间的口语练习**
- 周六下午，孩子在车上
- 打开小程序，选择"主题口语对话"
- 与AI教师进行10分钟"动物主题"英语对话
- AI实时纠正发音和语法
- 完成后获得积分，连续打卡天数+1

## 3. 功能需求

### 3.1 功能架构

```
AI智慧学伴
├── 家长端
│   ├── 家庭仪表盘
│   ├── 审批中心
│   ├── 学习简报
│   ├── 孩子管理
│   ├── 学习设置
│   └── 订阅管理
├── 孩子端
│   ├── 学习主页
│   ├── 英语学习（5个场景）
│   ├── 成就中心
│   └── 我的学习
└── 后台系统
    ├── AI记忆引擎
    ├── 智能推送调度
    ├── 内容管理
    └── 数据分析
```

### 3.2 核心功能详述

#### 3.2.1 家庭账户体系

**FR-001: 手机号登录注册**
- 用户通过手机号+验证码登录
- 首次登录自动创建家庭账户和家长账户
- 支持微信一键授权获取头像昵称

**验收标准**:
- [ ] 验证码60秒内有效
- [ ] 验证码发送频率限制（同一手机号1分钟内最多1次）
- [ ] 登录成功后返回JWT token
- [ ] Token有效期7天

**FR-002: 家庭成员管理**
- 主家长可邀请另一位家长加入家庭（通过手机号）
- 父母双方均可查看所有孩子的学习情况
- 父母双方均可审批学习推送

**验收标准**:
- [ ] 邀请链接24小时内有效
- [ ] 被邀请家长接受后自动加入家庭
- [ ] 家庭最多支持2位家长
- [ ] 双方权限完全相同

**FR-003: 孩子电子建档**
- 家长添加孩子基本信息：姓名、性别、出生日期、年级
- 系统根据年级自动判断K12阶段（低年级/中年级/高年级/初中）
- 自动创建孩子的全人化档案和游戏化档案

**验收标准**:
- [ ] 一个家庭最多添加5个孩子
- [ ] 年级范围1-9年级
- [ ] K12阶段自动映射：1-2年级→低年级、3-4→中年级、5-6→高年级、7-9→初中
- [ ] 创建孩子时同步创建ChildProfile和ChildGameProfile

#### 3.2.2 AI全人化记忆引擎

**FR-004: 学习数据层记忆**
- 记录每个孩子的学习进度、知识点掌握度、历史正确率
- 记录学习时长、学习频率、学习习惯

**FR-005: 认知风格层记忆**
- AI通过互动数据推断孩子的学习风格（视觉型/听觉型/动手型）
- 记录注意力持续时长、最佳学习时段
- 记录理解速度和记忆特点

**FR-006: 性格情感层记忆**
- AI分析孩子的性格特征（内向/外向、好奇心、抗挫折能力）
- 记录最有效的鼓励方式
- 记录兴趣爱好和话题偏好

**FR-007: 教学策略层记忆**
- AI为每个孩子总结最佳教学节奏
- 记录薄弱环节突破策略
- 记录已验证有效的教学方法

**验收标准**:
- [ ] 每次学习会话结束后自动更新档案
- [ ] AI分析结果以JSON格式存储在ChildProfile中
- [ ] 家长可查看孩子的全人化画像（可视化展示）
- [ ] AI教师每次互动前加载孩子的完整上下文

#### 3.2.3 英语学习 - 艾宾浩斯单词记忆

**FR-008: 新单词学习**
- AI根据孩子K12阶段推荐新单词
- 每个单词包含：英文、音标、中文含义、例句、图片联想
- AI教师用孩子感兴趣的方式讲解单词

**FR-009: 艾宾浩斯间隔复习**
- 标准复习间隔：5分钟→30分钟→12小时→1天→2天→4天→7天→15天
- AI根据孩子的遗忘模式动态调整间隔
- 答错的单词回退1-2个复习阶段

**FR-010: 多种互动方式**
- 中文含义选择/输入
- 英文单词拼写
- 语音跟读（发音评测）
- 看图说单词

**验收标准**:
- [ ] 每次学习会话包含复习单词+新单词
- [ ] 复习单词优先级高于新单词
- [ ] 每个单词的互动至少包含2种方式（如：含义+跟读）
- [ ] 答对进入下一阶段，答错回退
- [ ] 连续答对3次标记为"已掌握"
- [ ] AI教师每次回答包含：对错判断+学习指导+鼓励

#### 3.2.4 英语学习 - 发音/语音练习

**FR-011: 发音练习**
- AI标准发音示范
- 孩子跟读后上传录音
- 调用语音评测API获取发音评分（音标级别）
- AI教师基于评分给出纠音建议

**验收标准**:
- [ ] 支持单词跟读和句子跟读
- [ ] 发音评分0-100分
- [ ] 评分<60分时AI给出具体纠音建议
- [ ] 支持对比回放（AI发音 vs 孩子发音）

#### 3.2.5 英语学习 - 情景造句与对话

**FR-012: 情景造句**
- 基于当前学习的单词生成情景题目
- 难度梯度：填空→半开放造句→自由表达
- AI评估语法正确性和用词恰当性

**验收标准**:
- [ ] 低年级以填空为主
- [ ] 中高年级逐步增加开放度
- [ ] AI不直接给答案，而是引导纠正
- [ ] 错误时给出正确示范

#### 3.2.6 英语学习 - 分级阅读

**FR-013: 分级阅读**
- 根据孩子词汇量和阅读能力匹配阅读材料
- AI朗读文章（TTS）
- 孩子跟读或默读
- 阅读理解提问检验理解深度
- 生词自动纳入艾宾浩斯复习

**验收标准**:
- [ ] 阅读材料难度匹配孩子当前水平
- [ ] 文章包含3-5个理解问题
- [ ] 生词自动添加到单词库
- [ ] 阅读完成后生成阅读报告

#### 3.2.7 英语学习 - 主题式口语对话

**FR-014: 主题口语对话**
- 基于主题（家庭、学校、旅行、动物等）开展对话
- AI扮演不同角色与孩子对话
- 实时语法纠正融入对话
- 对话结束后生成能力评估

**验收标准**:
- [ ] 支持至少10个主题
- [ ] 对话轮次5-10轮
- [ ] AI根据孩子水平调整对话难度
- [ ] 语法错误时自然纠正（不打断对话流畅性）

#### 3.2.8 智能推送与家长审批

**FR-015: AI智能推送生成**
- 定时调度器每小时检查所有孩子的学习需求
- 检查是否在家长设置的时间窗口内
- 检查艾宾浩斯到期的复习需求
- 综合AI记忆引擎的分析生成推送建议
- 推送内容包含：学习情况总结+推送缘由+预期效果+具体内容

**FR-016: 家长审批流程**
- 家长收到待审批推送通知（小程序订阅消息）
- 家长查看推送详情
- 家长操作：确认/修改/拒绝/延后
- 确认后推送到孩子端

**验收标准**:
- [ ] 推送生成考虑时间窗口、记忆曲线、学习负荷
- [ ] 推送通知及时送达家长
- [ ] 家长可修改推送内容（如减少单词数量）
- [ ] 家长拒绝后AI记录原因，优化后续推送
- [ ] 延后推送可选择延后时间（1小时/3小时/明天）

#### 3.2.9 即时学习简报

**FR-017: 自动生成简报**
- 每次孩子完成学习后自动生成简报
- 简报内容：一句话总结+正确率+新学内容+进步点+需加强点+AI评语+鼓励语
- 推送通知给家长

**验收标准**:
- [ ] 学习会话结束后30秒内生成简报
- [ ] 简报推送给家长（小程序订阅消息）
- [ ] 家长可查看历史简报列表
- [ ] 简报支持按孩子、按日期筛选

#### 3.2.10 游戏化激励体系

**FR-018: 积分系统**
- 每次学习获得积分（基础分+正确率加成+连续打卡加成）
- 积分可用于兑换虚拟奖励（暂不支持实物兑换）

**FR-019: 徽章系统**
- 完成里程碑解锁徽章（如"单词大师100""连续学习7天"）
- 徽章分类：里程碑、掌握类、连续打卡、挑战类

**FR-020: 等级系统**
- 根据总积分计算等级（1-5级）
- 等级名称：学习新手→小小学者→知识探索者→学习达人→学霸

**FR-021: 连续打卡**
- 记录连续学习天数
- 中断后可用积分"补签"（限1次/周）

**FR-022: 每日挑战**
- 每天一道特殊挑战题
- 完成获得额外积分和徽章

**验收标准**:
- [ ] 积分实时更新
- [ ] 徽章解锁时弹窗提示+动画效果
- [ ] 等级提升时特殊庆祝动画
- [ ] 连续打卡中断时提示补签
- [ ] 每日挑战每天0点刷新

## 5. 用户体验流程

### 5.1 家长首次使用流程

```
1. 扫码进入小程序
2. 手机号+验证码登录
3. 授权微信头像昵称
4. 创建家庭账户
5. 添加第一个孩子信息
6. 选择订阅学科（英语）
7. 设置学习时间窗口
8. 完成引导，进入家庭仪表盘
```

### 5.2 孩子首次学习流程

```
1. 家长帮助孩子登录（选择孩子身份）
2. AI教师自我介绍
3. 简单的互动测试（了解孩子水平）
4. 开始第一次单词学习（3-5个简单单词）
5. 完成后获得首个徽章"初识英语"
6. 查看积分和等级
```

### 5.3 日常学习流程

```
家长端:
1. 收到推送通知"AI为小明推荐了今日学习内容"
2. 打开审批中心查看详情
3. 确认推送

孩子端:
4. 收到学习任务通知
5. 打开小程序，看到新任务
6. 开始学习（与AI教师互动）
7. 完成学习，获得积分和反馈
8. 查看成就进度

家长端:
9. 收到即时简报
10. 查看孩子学习情况
```

## 7. 约束与假设

### 7.1 技术约束

- 微信小程序平台限制（包大小、API限制等）
- LLM API调用成本和速度
- 语音识别准确率依赖第三方服务

### 7.2 业务约束

- Phase 1 仅支持英语学习
- 暂不支持实物奖励兑换
- 暂不支持家长间社交功能

### 7.3 假设

- 家长愿意为AI个性化教学付费
- 孩子能够独立使用小程序学习
- AI教师的互动能够保持孩子兴趣
- 家长审批不会成为学习的阻碍

## 9. 风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| LLM API成本过高 | 高 | 中 | 优化Prompt、缓存常见回复、探索国产LLM |
| 家长审批成为阻碍 | 中 | 中 | 提供"自动审批"选项、优化推送时机 |
| 孩子兴趣难以持续 | 高 | 高 | 强化游戏化、增加社交元素、定期更新内容 |
| 语音识别准确率不足 | 中 | 低 | 多家服务商对比、提供文字输入备选 |
| 竞品快速跟进 | 中 | 中 | 强化AI记忆差异化、建立用户迁移成本 |
